# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.4.x   | :white_check_mark: |
| < 1.4   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via:
- Email: security@ecostack.ee
- GitHub Security Advisory: [Create advisory](https://github.com/Deploy-Monster/DeployMonster_GO/security/advisories/new)

You should receive a response within 48 hours. If you don't, please follow up via email.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact
- Suggested fix (if any)

## Disclosure Policy

- We will acknowledge receipt within 48 hours
- We will confirm the vulnerability within 7 days
- We will release a fix within 30 days (critical) or 90 days (others)
- We will publicly disclose after the fix is released

## Security Best Practices

When using DeployMonster:
- Keep your installation updated
- Use strong admin passwords
- Enable 2FA for all accounts
- Configure firewall rules properly
- Keep Docker updated
- Regularly rotate API keys and secrets
